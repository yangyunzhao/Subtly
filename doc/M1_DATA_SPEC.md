# M1 数据字段与交互规范 / Data & Interaction Spec

> 目的：在 M1 阶段统一数据字段、单位、默认值与必填校验口径，便于后续页面实现与数据存储对齐。

## 1) 通用规范

- **日期格式**：统一使用 `YYYY-MM-DD`，通过系统日期选择器输入。
- **数字输入**：允许输入整数或 1 位小数（体重、腰围）。
- **空值处理**：未填写字段不保存，保持为空字符串或 `null`。
- **默认值**：表单加载时默认带出今天日期。

## 2) 健身记录 / FitnessEntry

| 字段 | 类型 | 单位 | 必填 | 默认值 | 交互说明 |
| --- | --- | --- | --- | --- | --- |
| date | string | - | ✅ | 今天 | 日期选择器，格式 `YYYY-MM-DD`。 |
| type | string | - | ✅ | running | 训练类型（跑步、骑车、力量、平板支撑、俯卧撑、其他）。 |
| duration | number | 分钟 | ⚠️ 条件必填 | - | 时间类项目建议填写，若 `count` 为空则需要填写。 |
| count | number | 次 | ⚠️ 条件必填 | - | 次数类项目建议填写，若 `duration` 为空则需要填写。 |
| calories | number | 千卡 | 否 | 0 | 记录热量消耗，单位千卡。 |
| notes | string | - | 否 | 空 | 备注说明。 |

**条件必填规则**：`duration` 与 `count` 至少填写一个，用于覆盖不同类型训练的记录方式。

## 3) 身体数据 / BodyMetrics

| 字段 | 类型 | 单位 | 必填 | 默认值 | 交互说明 |
| --- | --- | --- | --- | --- | --- |
| date | string | - | ✅ | 同健身记录日期 | 与当日健身记录一致。 |
| weight | number | 斤 | ⚠️ 条件必填 | - | 保留 1 位小数。 |
| waistline | number | cm | ⚠️ 条件必填 | - | 保留 1 位小数。 |

**条件必填规则**：`weight` 与 `waistline` 至少填写一个，便于记录当日身体数据。

## 4) 展示口径

- 列表项展示默认包含：日期 + 训练摘要 + 身体数据摘要。
- 空状态文案建议包含：**未记录提示 + 引导按钮**。
